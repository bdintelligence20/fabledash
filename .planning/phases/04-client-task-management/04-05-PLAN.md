  <name>Task 1: Build Kanban board component with drag-and-drop status transitions</name>
  <files>src/components/tasks/KanbanBoard.tsx</files>
  <action>
Create a Kanban board component that displays tasks in status columns with drag-and-drop:

**Component props:**
- tasks: TaskResponse[] (full task list, component groups them by status)
- clientMap: Record&lt;string, string&gt; (client id to name mapping)
- onStatusChange: (taskId: string, newStatus: TaskStatus) => Promise&lt;void&gt; (callback for status update)
- onTaskClick: (taskId: string) => void (navigate to task detail)

**Layout — 5 status columns:**
- Columns in workflow order: Todo, In Progress, In Review, Done, Blocked
- Each column has:
  - Column header: status label + task count Badge
  - Column background: subtle color tint matching status (e.g., surface-50 with colored left border)
  - Scrollable card list within the column
- Responsive: horizontal scroll on mobile (flex-nowrap overflow-x-auto), all 5 visible on desktop (min-w-[200px] per column)

**Task cards within columns:**
- Card component (compact padding) showing:
  - Task title (bold, clickable — calls onTaskClick)
  - Client name (text-sm text-surface-500)
  - Priority Badge (small size)
  - Due date (text-xs, red if overdue)
  - Comment count icon + number (if comments > 0)
- Card has subtle hover effect (shadow-card-hover on hover)

**Drag and drop — use native HTML5 drag and drop API (no external library):**
- Cards are draggable (draggable="true" attribute)
- Columns are drop targets (onDragOver, onDrop handlers)
- On drag start: store task ID and source status in dataTransfer
- On drag over column: add visual highlight to column (border-dashed, bg tint change)
- On drop: extract task ID from dataTransfer, call onStatusChange(taskId, columnStatus)
- On drag end: remove all visual highlights
- Keep it simple — no reordering within columns, only cross-column status changes
- Provide cursor feedback: cursor-grab on cards, cursor-grabbing while dragging

**Column color scheme (left border + subtle background):**
- Todo: surface/gray (border-l-4 border-surface-400, bg-surface-50)
- In Progress: primary/blue (border-l-4 border-primary-400, bg-primary-50)
- In Review: warning/amber (border-l-4 border-warning-400, bg-warning-50)
- Done: success/green (border-l-4 border-success-400, bg-success-50)
- Blocked: danger/red (border-l-4 border-danger-400, bg-danger-50)

**Empty column state:**
- Show "No tasks" centered text in empty columns
- Column still accepts drops even when empty

**Important:**
- No external drag-and-drop library (react-beautiful-dnd, dnd-kit, etc.) — use native HTML5 API to avoid new dependencies
- Component is purely presentational + drag handlers; parent (TasksPage) owns data fetching and API calls
- Use Tailwind classes from design system, no inline styles

**Part A: CalendarView component (src/components/tasks/CalendarView.tsx)**

Build a monthly calendar view showing tasks plotted on their due dates:

**Component props:**
- tasks: TaskResponse[] (filtered to only those with due_date)
- clientMap: Record&lt;string, string&gt;
- onTaskClick: (taskId: string) => void

**Layout — month grid:**
- Header row: month/year display with prev/next month navigation buttons (< and > Button icons)
- Day-of-week header row: Mon, Tue, Wed, Thu, Fri, Sat, Sun
- 6-row grid of day cells (7 columns), covering the full calendar month
- Current month state managed internally (useState with current month/year)

**Day cells:**
- Show day number in corner (text-xs text-surface-400 for non-current-month days, text-surface-700 for current month)
- Today highlighted with primary ring or bg-primary-50
- Tasks with due_date matching the day rendered as compact pills:
  - Pill shows: truncated task title (max ~20 chars)
  - Pill color matches status (same color mapping as Kanban)
  - Pill is clickable (calls onTaskClick)
  - Max 3 visible tasks per day, then "+N more" overflow indicator
- Empty days show nothing

**Calendar computation (no external date library):**
- Calculate first day of month (new Date(year, month, 1).getDay())
- Calculate days in month (new Date(year, month + 1, 0).getDate())
- Fill leading/trailing days from prev/next month (grayed out)
- Group tasks by date string (YYYY-MM-DD) for O(1) lookup per day cell

**Responsive:**
- On mobile: day cells show dot indicators instead of task titles (colored dots by status)
- "List view for this day" on mobile tap — show tasks in a small dropdown or expand
