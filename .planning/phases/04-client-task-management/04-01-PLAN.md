  <action>
Replace the placeholder ClientsPage with a full client management page:

**Data fetching:**
- Use apiClient.get to fetch from /clients endpoint on mount and after mutations
- Store clients in local state; manage loading and error states
- Type the response as { success: boolean; data: ClientResponse[] } where ClientResponse matches the backend model (id, name, partner_group, contact_email, contact_phone, description, is_active, created_at, updated_at, created_by)

**Layout and controls:**
- Page header: "Clients" title with a "New Client" Button (variant="primary") that opens the create modal
- Filter bar row below header:
  - Text Input for search (filters client name client-side via .filter())
  - Select dropdown for partner group filter (options: All, Collab, EDCP, Direct Clients, Separate Businesses) — when selected, re-fetches from /clients?partner_group={value}
  - Badge-style count showing total clients displayed

**Data table:**
- Use the Table compound component (Table, Table.Head, Table.Body, Table.Row, Table.HeaderCell, Table.Cell)
- Columns: Name, Partner Group (as Badge with color per group — primary for Collab, secondary for EDCP, success for Direct, accent for Separate), Contact Email, Status (Badge: green "Active" / gray "Inactive"), Actions
- Actions column: Edit button (opens modal pre-filled), Deactivate/Activate toggle button
- Empty state: centered message "No clients found" when list is empty
- Loading state: Spinner component centered in table area

**Partner group colors mapping:**
- collab -> "primary" badge variant
- edcp -> "secondary" badge variant
- direct_clients -> "success" badge variant
- separate_businesses -> "warning" badge variant

**Create/Edit Modal (inline in same file or extracted as local component):**
- Modal component with title "New Client" or "Edit Client" depending on mode
- Form fields: name (Input, required), partner_group (Select, required, options from PartnerGroup enum values), contact_email (Input), contact_phone (Input), description (Input or textarea)
- Submit: POST /clients for create, PUT /clients/{id} for edit via apiClient
- On success: close modal, refresh client list
- On error: display error message in modal
- Cancel button closes modal without saving

**Important patterns:**
- Import UI components from ../components/ui (relative import, no @/ alias — decision from 03-04)
- Use apiClient from ../lib/api for all API calls
- No external state management library — local useState/useEffect only
- Partner group display names: collab -> "Collab", edcp -> "EDCP", direct_clients -> "Direct Clients", separate_businesses -> "Separate Businesses"
