    1. Add `openpyxl` to python-backend/requirements.txt (for .xlsx parsing). csv is stdlib.

    2. Create python-backend/app/utils/excel_parser.py:

       Function parse_pnl_file(file_content: bytes, filename: str) -> list[PnlRow]:
       - Detect file type from filename extension (.csv, .xlsx, .xls)
       - For CSV: use csv.DictReader with io.StringIO
       - For Excel: use openpyxl.load_workbook with io.BytesIO
       - Expected column pattern (flexible matching — case-insensitive, partial match):
         - "category" or "account" or "line item" → category
         - "subcategory" or "sub-category" or "detail" → subcategory (optional)
         - "actual" or "actuals" or "amount" → actual
         - "forecast" or "budget" or "projected" → forecast (optional)
         - "period" or "month" or "date" → period (optional, defaults to filename hint or current month)
       - Parse each row into PnlRow model
       - Calculate variance = actual - forecast when both present
       - Skip rows where category is empty or looks like a header/total row
       - Return list of PnlRow
       - Raise ValueError with descriptive message if file format is unrecognizable

       Function parse_forecast_file(file_content: bytes, filename: str) -> list[dict]:
       - Same file type detection
       - More flexible — just extract all rows as dicts with their column headers
       - Return raw list of row dicts (forecast formats vary more than P&L)

    3. Create python-backend/app/api/financial_uploads.py:
       - router = APIRouter()
       - All endpoints require Depends(get_current_user)

       Endpoints:

       a. POST /pnl — Upload P&L report (CEO only):
          - Accepts multipart file upload (UploadFile)
          - Optional query param: period (str, e.g., "2026-01" — defaults to current month)
          - Read file content
          - Parse with parse_pnl_file()
          - Create PnlUpload document in Firestore with rows, filename, period, uploaded_by
          - Return: { success: true, data: { id: str, row_count: int, period: str } }
          - On parse error: return 400 with descriptive error

       b. GET /pnl — List uploaded P&L reports:
          - Query params: period (optional), limit (int, default 12)
          - Query Firestore PNL_COLLECTION
          - Return: { success: true, data: list of { id, filename, period, row_count, uploaded_at } } (without full rows — summary only)

       c. GET /pnl/{upload_id} — Get full P&L upload with rows:
          - Fetch document from Firestore
          - Return: { success: true, data: PnlUpload with all rows }

       d. DELETE /pnl/{upload_id} — Delete a P&L upload (CEO only):
          - Delete from Firestore
          - Return: { success: true, message: "P&L report deleted" }

    4. Update python-backend/app/main.py:
       - Import and include financial_uploads_router with prefix="/financial", tags=["financial"]
