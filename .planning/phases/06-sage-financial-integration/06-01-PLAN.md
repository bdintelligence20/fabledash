    4. Create python-backend/app/utils/sage_client.py:
       - Import httpx, logging, datetime
       - Import get_settings from app.config
       - Import get_firestore_client from app.utils.firebase_client
       - Import SageCredentials, SAGE_CREDENTIALS_COLLECTION from app.models.financial

       Class SageClient:
       - __init__(self): stores settings, base_url, creates httpx.AsyncClient
       - async get_credentials(self) -> SageCredentials | None: Read stored credentials from Firestore (document ID: "current")
       - async save_credentials(self, creds: SageCredentials): Save to Firestore
       - async refresh_token(self) -> SageCredentials: POST to Sage token endpoint with refresh_token grant, save new tokens, return updated creds
       - async _ensure_valid_token(self) -> str: Get credentials, check expires_at, refresh if expired, return access_token
       - async get(self, endpoint: str, params: dict | None = None) -> dict: GET request to Sage API with auth header. Auto-refresh on 401.
       - async get_paginated(self, endpoint: str, params: dict | None = None) -> list[dict]: Handle Sage pagination (follows $next links)
       - async exchange_code(self, code: str) -> SageCredentials: Exchange OAuth2 authorization code for tokens
       - is_connected(self) -> bool: Check if valid credentials exist

       Helper function: get_sage_client() -> SageClient (cached singleton pattern)

       - All endpoints require Depends(get_current_user), CEO-only endpoints use Depends(require_ceo)

       Endpoints:

       a. GET /status — Check Sage connection status (any authenticated user)
          - Returns: { success: true, data: { connected: bool, last_sync: str | null } }
          - Reads credentials from Firestore, checks if valid

       b. GET /connect — Initiate OAuth2 flow (CEO only)
          - Builds Sage authorization URL with client_id, redirect_uri, scope, response_type=code
          - Returns: { success: true, data: { authorization_url: str } }
          - Scope should include: full_access (Sage Business Cloud Accounting scope)

       c. GET /callback — OAuth2 callback handler (no auth — Sage redirects here)
          - Receives ?code=... query parameter
          - Calls sage_client.exchange_code(code)
          - Stores credentials in Firestore
          - Returns HTML page with "Connected successfully! You can close this window." or redirects to frontend

       d. POST /disconnect — Remove Sage credentials (CEO only)
          - Deletes credentials document from Firestore
          - Returns: { success: true, message: "Sage disconnected" }

       e. POST /sync — Trigger manual sync (CEO only)
          - Placeholder for now — returns { success: true, message: "Sync started" }
