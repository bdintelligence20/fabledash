       Class TranscriptProcessor:
       - __init__: loads OpenAI client from config OPENAI_API_KEY

       - async extract_entities(transcript_text: str) -> dict:
         Use OpenAI to extract: client names, project/task references, people mentioned, dates discussed, action items
         Return structured dict of extracted entities

       - async match_entities_to_records(entities: dict) -> dict:
         Match extracted client names to Firestore clients (fuzzy match)
         Match task references to Firestore tasks
         Return { matched_client_id, matched_task_ids, confidence_scores }

       - async generate_summary(transcript_text: str, meeting_title: str) -> str:
         Use OpenAI to generate a concise meeting summary (3-5 bullet points)
         Include: key decisions, discussed topics, next steps

       - async extract_action_items(transcript_text: str) -> list[str]:
         Use OpenAI to extract action items from transcript
         Return list of action item strings

       - async process_transcript(meeting_id: str, transcript_text: str) -> dict:
         Orchestrate: extract_entities → match_entities → generate_summary → extract_action_items
         Update meeting document in Firestore with results
         Return processing results

       Use OpenAI gpt-4o-mini for all calls (cost-effective for extraction tasks).
       Handle missing OPENAI_API_KEY gracefully (skip AI processing, return empty results).
