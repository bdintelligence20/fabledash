    Add to the existing financial_data router:

    GET /cost-benefit — Client value analysis (ZAR per hour):
    - Query params: date_from (optional), date_to (optional)
    - Logic:
      1. Fetch all clients from Firestore
      2. Fetch all invoices within date range (paid status) — group revenue by client_id
      3. Fetch all time logs within date range — group hours by client_id
      4. For each client: calculate zar_per_hour = total_revenue / total_hours (handle div by zero)
      5. Identify pass-through: clients where revenue > 0 but hours are very low (< 2 hrs) or zero — these are pass-through projects
      6. Sort by zar_per_hour descending
    - Return:
      ```json
      {
        "success": true,
        "data": {
          "clients": [
            {
              "client_id": "c1",
              "client_name": "Acme Corp",
              "partner_group": "collab",
              "total_revenue": 50000,
              "total_hours": 120.5,
              "zar_per_hour": 414.94,
              "invoice_count": 5,
              "is_pass_through": false
            }
          ],
          "summary": {
            "total_revenue": 250000,
            "total_hours": 500,
            "average_zar_per_hour": 500,
            "pass_through_count": 2,
            "pass_through_revenue": 30000
          }
        }
      }
      ```

    Use get_firestore_client() for all Firestore queries. Import INVOICES_COLLECTION from financial models.
