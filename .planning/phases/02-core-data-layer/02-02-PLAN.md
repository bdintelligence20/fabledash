<task type="auto">
  <name>Task 1: Create client API router with full CRUD</name>
  <files>python-backend/app/api/clients.py</files>
  <action>
    1. Create python-backend/app/api/clients.py:
       - router = APIRouter()
       - All endpoints use `user: CurrentUser = Depends(get_current_user)` for auth
       - Use `get_firestore_client()` from app.utils.firebase_client for Firestore access
       - Use COLLECTION_NAME from app.models.client

       Endpoints:

       a. POST / — Create client
          - Body: ClientCreate
          - Set created_at, updated_at to datetime.utcnow()
          - Set created_by to user.uid
          - Convert model to dict, add timestamps and created_by
          - db.collection(COLLECTION_NAME).add(doc_dict) — let Firestore generate ID
          - Return BaseResponse with success=True and the new client data (include Firestore-generated ID)

       b. GET / — List clients
          - Optional query params: partner_group (PartnerGroup | None), is_active (bool | None = True)
          - Query Firestore: start with db.collection(COLLECTION_NAME)
          - Apply .where("partner_group", "==", partner_group.value) if provided
          - Apply .where("is_active", "==", is_active) if provided (default filters to active only)
          - Order by created_at descending
          - Stream results, convert each doc to ClientResponse (doc.to_dict() + id=doc.id)
          - Return {"success": true, "data": [...clients]}

       c. GET /{client_id} — Get single client
          - Fetch doc by ID: db.collection(COLLECTION_NAME).document(client_id).get()
          - If not exists, raise HTTPException 404 "Client not found"
          - Return {"success": true, "data": client}

       d. PUT /{client_id} — Update client
          - Body: ClientUpdate
          - Fetch existing doc first to verify it exists (404 if not)
          - Use model.model_dump(exclude_none=True) to get only provided fields
          - Add updated_at = datetime.utcnow() to update dict
          - db.collection(COLLECTION_NAME).document(client_id).update(update_dict)
          - Return updated client

       e. DELETE /{client_id} — Delete client (soft delete: set is_active=False)
          - Verify doc exists (404 if not)
          - Update is_active=False and updated_at
          - Return BaseResponse with success=True, message="Client deactivated"
          - WHY soft delete: clients are referenced by tasks and time logs; hard delete would orphan those references

       Error handling:
       - Wrap Firestore operations in try/except
       - Return ErrorResponse for failures
       - Use proper HTTP status codes (400, 404, 500)

       Datetime handling:
       - Firestore returns datetime objects natively — use them directly
       - For create/update, use datetime.utcnow()
       - When converting Firestore docs to Pydantic models, handle the id field (Firestore stores it separately from doc data)
