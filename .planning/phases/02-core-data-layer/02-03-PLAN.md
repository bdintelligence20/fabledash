  <name>Task 1: Create task API router with CRUD and filtering</name>
  <files>python-backend/app/api/tasks.py</files>
  <action>
    1. Create python-backend/app/api/tasks.py:
       - router = APIRouter()
       - All endpoints use `user: CurrentUser = Depends(get_current_user)` for auth
       - Use `get_firestore_client()` for Firestore access
       - Use COLLECTION_NAME from app.models.task

       Core CRUD endpoints:

       a. POST / — Create task
          - Body: TaskCreate
          - Set created_at, updated_at to datetime.utcnow()
          - Set created_by to user.uid
          - Initialize comments=[], attachments=[]
          - Convert model to dict, add timestamps/created_by/empty arrays
          - Convert enum values to .value for Firestore storage (status, priority)
          - db.collection(COLLECTION_NAME).add(doc_dict)
          - Return {"success": true, "data": task_with_id}

       b. GET / — List tasks with filtering
          - Optional query params:
            - client_id: str | None — filter by client
            - status: TaskStatus | None — filter by status
            - priority: TaskPriority | None — filter by priority
            - assigned_to: str | None — filter by assignee uid
          - Build Firestore query with chained .where() for each provided filter
          - Order by created_at descending
          - Stream results, convert to TaskResponse list
          - Return {"success": true, "data": [...tasks]}

       c. GET /{task_id} — Get single task
          - Fetch by ID, 404 if not found
          - Return {"success": true, "data": task}

       d. PUT /{task_id} — Update task
          - Body: TaskUpdate
          - Verify task exists (404 if not)
          - model_dump(exclude_none=True) for partial update
          - Convert enum values to .value for any status/priority in update
          - Add updated_at = datetime.utcnow()
          - db.collection(COLLECTION_NAME).document(task_id).update(update_dict)
          - Re-fetch and return updated task

       e. DELETE /{task_id} — Delete task (hard delete)
          - Verify exists (404 if not)
          - db.collection(COLLECTION_NAME).document(task_id).delete()
          - Return BaseResponse success=True, message="Task deleted"
          - WHY hard delete (not soft): tasks are leaf entities not referenced by other collections; deactivated tasks add query noise

       Comment sub-resource endpoints:

       f. POST /{task_id}/comments — Add comment
          - Body: {content: str}
          - Generate comment id with uuid4().hex[:12]
          - Build TaskComment with author_uid=user.uid, author_name=user.display_name, created_at=utcnow()
          - Use Firestore ArrayUnion to append comment to comments array
          - Return {"success": true, "data": comment}

       g. DELETE /{task_id}/comments/{comment_id} — Remove comment
          - Fetch task, find comment in comments array
          - 404 if comment not found
          - Use Firestore ArrayRemove to remove the matching comment dict
          - Return BaseResponse success=True

       Attachment sub-resource endpoints:

       h. POST /{task_id}/attachments — Add attachment metadata
          - Body: {filename: str, url: str, content_type: str | None}
          - Generate attachment id with uuid4().hex[:12]
          - Build TaskAttachment with uploaded_by=user.uid, uploaded_at=utcnow()
          - Use Firestore ArrayUnion to append
          - Return {"success": true, "data": attachment}
          - NOTE: This stores metadata only. Actual file upload is a future concern (Phase 9 document processing).

       i. DELETE /{task_id}/attachments/{attachment_id} — Remove attachment
          - Same pattern as comment deletion using ArrayRemove
          - Return BaseResponse success=True

       Firestore helpers within the module:
       - Create a _doc_to_task(doc) helper that converts a Firestore document snapshot to TaskResponse dict (handles doc.id -> id mapping and nested comment/attachment reconstruction)
       - Handle Firestore DatetimeWithNanoseconds -> Python datetime conversion if needed

       Error handling:
       - Wrap all Firestore operations in try/except
       - 404 for missing tasks/comments/attachments
       - 400 for invalid input
       - Return ErrorResponse for failures
