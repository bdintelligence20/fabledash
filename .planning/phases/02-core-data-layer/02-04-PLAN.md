  <name>Task 1: Create time log API router with CRUD and auto-duration</name>
  <files>python-backend/app/api/time_logs.py</files>
  <action>
    1. Create python-backend/app/api/time_logs.py:
       - router = APIRouter()
       - All endpoints use `user: CurrentUser = Depends(get_current_user)` for auth
       - Use `get_firestore_client()` for Firestore access
       - Use COLLECTION_NAME from app.models.time_log
       - Import calculate_duration_minutes from app.models.time_log

       Endpoints:

       a. POST / — Create time log
          - Body: TimeLogCreate
          - Validate end_time > start_time, return 400 if not (with clear message: "End time must be after start time")
          - Calculate duration_minutes using calculate_duration_minutes(start_time, end_time)
          - Set created_at, updated_at to datetime.utcnow()
          - Set created_by to user.uid
          - Convert model to dict:
            - date field: convert date to ISO string for Firestore storage (date.isoformat())
            - start_time/end_time: convert time to string (time.isoformat()) for Firestore storage
            - Add duration_minutes, created_at, updated_at, created_by
          - db.collection(COLLECTION_NAME).add(doc_dict)
          - Return {"success": true, "data": time_log_with_id}

       b. GET / — List time logs with filtering
          - Optional query params:
            - client_id: str | None — filter by client
            - task_id: str | None — filter by task
            - date_from: date | None — filter logs on or after this date
            - date_to: date | None — filter logs on or before this date
            - created_by: str | None — filter by user who created the entry
          - Build Firestore query with chained .where() for each filter
          - For date range: use .where("date", ">=", date_from.isoformat()) and .where("date", "<=", date_to.isoformat())
          - Order by date descending, then start_time descending
          - Stream results, convert to TimeLogResponse list
          - Return {"success": true, "data": [...time_logs]}

       c. GET /{time_log_id} — Get single time log
          - Fetch by ID, 404 if not found
          - Return {"success": true, "data": time_log}

       d. PUT /{time_log_id} — Update time log
          - Body: TimeLogUpdate
          - Verify exists (404 if not)
          - model_dump(exclude_none=True) for partial update
          - If start_time or end_time is being updated:
            - Get the other time from existing doc if not provided in update
            - Recalculate duration_minutes with the new/existing start and end times
            - Validate end > start (400 if not)
            - Add recalculated duration_minutes to update dict
          - Convert date/time fields to ISO strings for Firestore
          - Add updated_at = datetime.utcnow()
          - db.collection(COLLECTION_NAME).document(time_log_id).update(update_dict)
          - Re-fetch and return updated time log

       e. DELETE /{time_log_id} — Delete time log (hard delete)
          - Verify exists (404 if not)
          - db.collection(COLLECTION_NAME).document(time_log_id).delete()
          - Return BaseResponse success=True, message="Time log deleted"
          - WHY hard delete: time logs are leaf entities; incorrect entries should be removable without leaving ghost data

       Firestore helpers within the module:
       - Create a _doc_to_time_log(doc) helper that converts Firestore doc to TimeLogResponse dict
       - Handle date/time string -> Python date/time conversion when reading from Firestore
       - Handle duration_minutes inclusion in response

       Error handling:
       - Wrap all Firestore operations in try/except
       - Validate end_time > start_time on create and update (400 error)
       - 404 for missing time logs
       - Return ErrorResponse for failures
