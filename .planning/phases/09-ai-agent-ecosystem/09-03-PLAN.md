    1. Create python-backend/app/utils/vector_store.py:
       Uses OpenAI embeddings API (text-embedding-3-small model) for embedding generation.

       Class VectorStore:
       - __init__: OpenAI client from config
       - async generate_embedding(text: str) -> list[float]: Call OpenAI embeddings API
       - async store_embedding(chunk_id: str, document_id: str, embedding: list[float], content: str, metadata: dict): Store in Firestore "embeddings" collection
       - async search(query: str, agent_id: str|None = None, client_id: str|None = None, top_k: int = 5) -> list[dict]:
         Generate query embedding, then do brute-force cosine similarity against stored embeddings (Firestore doesn't have native vector search, so fetch all embeddings for the agent/client scope and compute similarity in Python). Return top_k results with content and score.
       - _cosine_similarity(a: list[float], b: list[float]) -> float: numpy dot product
       - async index_document_chunks(document_id: str, chunks: list[dict]): Generate embeddings for all chunks and store

       Note: For production scale, this should use a proper vector DB. For now, Firestore + in-memory similarity is fine for small document sets per agent.

    2. Create python-backend/app/utils/rag_engine.py:
       Class RAGEngine:
       - __init__: vector_store, openai_client
       - async retrieve_context(query: str, agent_id: str|None, client_id: str|None, top_k: int = 5) -> str:
         Search vector store, combine top results into context string
       - async generate_response(query: str, context: str, system_prompt: str|None, model: str = "gpt-4o-mini") -> str:
         Build messages with system prompt + context + user query, call OpenAI chat completions
       - async query(query: str, agent_id: str|None = None, client_id: str|None = None, system_prompt: str|None = None, model: str = "gpt-4o-mini") -> dict:
         Orchestrate: retrieve_context → generate_response. Return { answer, sources: [...] }
       - get_rag_engine() singleton
