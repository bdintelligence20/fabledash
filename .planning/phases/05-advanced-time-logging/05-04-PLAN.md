  <name>Task 1: Create time allocation aggregation endpoint</name>
  <files>python-backend/app/api/time_logs.py</files>
  <action>
    Add a new GET endpoint to the existing time_logs router:

    **GET /time-logs/allocation** — Aggregate time by partner group:
    - Query params: date_from (date, optional), date_to (date, optional)
    - Logic:
      1. Fetch all time logs within the date range (reuse existing Firestore query pattern from list endpoint)
      2. Fetch all clients from the clients collection to build a Map<client_id, partner_group>
      3. Group time logs by their client's partner_group
      4. For each group, calculate:
         - total_minutes: sum of duration_minutes
         - total_hours: total_minutes / 60 (float, 1 decimal)
         - billable_minutes: sum of duration_minutes where is_billable == True
         - non_billable_minutes: total - billable
         - entry_count: number of time log entries
         - percentage: (group total_minutes / grand total_minutes) * 100
      5. Return all four groups (even if zero): collab, edcp, direct_clients, separate_businesses
    - Response shape:
      ```json
      {
        "success": true,
        "data": {
          "period": { "from": "2026-01-01", "to": "2026-02-25" },
          "total_hours": 120.5,
          "groups": [
            {
              "partner_group": "collab",
              "total_hours": 45.5,
              "billable_hours": 40.0,
              "non_billable_hours": 5.5,
              "entry_count": 32,
              "percentage": 37.8
            }
          ]
        }
      }
      ```

    IMPORTANT: Place this route BEFORE the `/{time_log_id}` GET route in the router to avoid FastAPI treating "allocation" as a time_log_id path parameter. FastAPI matches routes in order.

    Use the existing `get_firestore_client`, `get_current_user` dependency, and `COLLECTION_NAME` from time_log models. Import `client.COLLECTION_NAME as CLIENT_COLLECTION` and `PartnerGroup` from client models.
