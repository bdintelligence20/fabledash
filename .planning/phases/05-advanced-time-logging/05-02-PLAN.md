  <files>src/components/time/RunningTimer.tsx, src/components/time/ActivityLog.tsx</files>
  <action>
    Create two new components in `src/components/time/`:

    **RunningTimer.tsx** — A persistent timer bar for tracking work in progress:
    - State: isRunning (bool), startedAt (Date | null), elapsedSeconds (number), description (string), selectedClientId (string), selectedTaskId (string | null), isBillable (bool)
    - When started: record current time as startedAt, start a setInterval(1000) incrementing elapsedSeconds
    - Display: elapsed time formatted as "HH:MM:SS" (e.g., "01:23:45"), pulsing green dot when running
    - Controls:
      - Description input (compact, inline — placeholder "What are you working on?")
      - Client select (compact — same client list pattern as 05-01 form, passed as prop)
      - Task select (optional, compact — same task list pattern)
      - Billable toggle (small checkbox)
      - Start/Stop button: green "Start" when stopped, red "Stop" when running
    - On Stop: call an `onStop` callback prop with { description, clientId, taskId, isBillable, startTime (HH:MM from startedAt), endTime (HH:MM from now) }. The parent (TimePage) will POST to /time-logs. Reset timer state after successful save.
    - Cleanup: clear interval on unmount via useEffect cleanup
    - Styling: Card with subtle colored left border (green when running, gray when idle). Compact single-row layout on desktop, stacked on mobile.

    **ActivityLog.tsx** — Chronological daily activity feed (Gemini-style):
    - Props: entries (array of time log responses for today), clients (Map<id, name>), tasks (Map<id, name>), loading (boolean)
    - Display: vertical timeline with each entry as a card-like row
    - Each entry shows:
      - Time range: "09:00 - 10:30" with duration badge ("1h 30m")
      - Client name (resolved from clients map)
      - Description text
      - Billable badge (green "Billable" or gray "Non-billable")
    - Timeline visual: vertical line on the left with small dots at each entry (use Tailwind border-l + relative positioning)
    - Group header: "Today — [date formatted as 'Tuesday, 25 Feb 2026']" with total hours for the day
    - Empty state: "No activity logged today. Start the timer or add a manual entry."
    - Loading state: Spinner component

    Import pattern: relative imports from ../../components/ui. Accept clients/tasks as props (fetched by parent).
