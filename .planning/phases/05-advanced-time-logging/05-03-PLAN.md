  <name>Task 1: Build TimeLogListPage with filters and data table</name>
  <files>src/pages/TimeLogListPage.tsx</files>
  <action>
    Create a new page component `TimeLogListPage` that displays time logs in a filterable, sortable table.

    **Filter bar** (horizontal row of controls at top, inside a Card):
    - Date range: two Input type="date" fields (From / To), default to current month (first day to today)
    - Client filter: Select dropdown fetched from GET /clients, "All Clients" as default/empty option
    - Task filter: Select dropdown fetched from GET /tasks (filtered by selected client_id if one is chosen), "All Tasks" as default
    - Partner group filter: Select with hardcoded options — "All Groups", "Collab", "EDCP", "Direct Clients", "Separate Businesses". Since the /time-logs endpoint doesn't filter by partner group directly, client-side filter by matching client_id to client's partner_group from the fetched clients list.
    - "Apply" Button to trigger fetch with current filter params

    **Data table** (using Table compound component from UI lib):
    - Columns: Date, Client, Task, Description, Start, End, Duration, Billable
    - Date: formatted as DD MMM YYYY (e.g., "25 Feb 2026")
    - Client: resolve client_id to client name from fetched clients (keep a Map<id, name>)
    - Task: resolve task_id to task name from fetched tasks (keep a Map<id, name>), show "—" if null
    - Duration: format duration_minutes as "Xh Ym" (e.g., "1h 30m" for 90 minutes)
    - Billable: Badge component — green "Billable" or gray "Non-billable"
    - Sort by date descending by default (API returns this order)

    **Summary row** below the table:
    - Total entries count
    - Total hours (sum of duration_minutes / 60, formatted to 1 decimal)
    - Total billable hours vs non-billable hours

    **Empty state**: Show a centered message "No time entries found" with a muted description when the list is empty.

    **Loading state**: Show Spinner component while fetching data.

    Data fetching:
    - Fetch time logs from GET /time-logs with query params: client_id, task_id, date_from, date_to (all optional)
    - Fetch clients from GET /clients on mount (for name resolution and partner group filtering)
    - Fetch tasks from GET /tasks on mount (for name resolution)
    - Use apiClient from src/lib/api.ts

    Import pattern: relative imports (../components/ui, ../lib/api). No @/ aliases.
  </action>
