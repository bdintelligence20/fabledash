  <name>Task 1: Create Firebase Auth middleware and dependencies</name>
  <files>python-backend/app/middleware/__init__.py, python-backend/app/middleware/auth.py, python-backend/app/dependencies/__init__.py, python-backend/app/dependencies/auth.py, python-backend/app/models/user.py</files>
  <action>
    1. Create python-backend/app/middleware/__init__.py (empty)

    2. Create python-backend/app/dependencies/__init__.py (empty)

    3. Create python-backend/app/models/user.py:
       - Define UserRole enum: CEO = "ceo", TEAM_MEMBER = "team_member"
       - Define CurrentUser(BaseModel) with uid: str, email: str | None, display_name: str | None, role: UserRole, custom_claims: dict | None = None

    4. Create python-backend/app/dependencies/auth.py:
       - Import firebase_admin.auth
       - Create `get_current_user(request: Request) -> CurrentUser` FastAPI dependency:
         a. Extract Bearer token from Authorization header
         b. If no token, raise HTTPException 401 "Missing authentication token"
         c. Verify token with firebase_admin.auth.verify_id_token(token)
         d. Extract uid, email, name from decoded token
         e. Check custom claims for role (default to "team_member" if no role claim)
         f. Return CurrentUser instance
         g. Catch firebase_admin.auth.InvalidIdTokenError → 401
         h. Catch firebase_admin.auth.ExpiredIdTokenError → 401 "Token expired"
         i. Catch firebase_admin.auth.RevokedIdTokenError → 401 "Token revoked"
       - Create `require_role(role: UserRole)` dependency factory:
         a. Returns a dependency that calls get_current_user then checks user.role
         b. If role doesn't match, raise HTTPException 403 "Insufficient permissions"
       - Create `require_ceo` = require_role(UserRole.CEO) shortcut

    5. Create python-backend/app/middleware/auth.py:
       - Create AuthMiddleware class (not used globally — auth is opt-in via dependencies)
       - Instead, create a list of PUBLIC_PATHS = ["/", "/health", "/docs", "/openapi.json", "/redoc"]
       - This file exists for reference but the actual enforcement is via FastAPI dependencies (Depends(get_current_user)) on protected routes
  </action>
  <verify>cd python-backend && python -c "from app.dependencies.auth import get_current_user, require_ceo; from app.models.user import CurrentUser, UserRole; print('Auth deps loaded')" && cd ..</verify>
  <done>CurrentUser model with role enum. get_current_user dependency extracts and verifies Firebase tokens. require_role/require_ceo dependency factories for role-based access. All importable without errors.</done>
</task>

<task type="auto">

  <files>python-backend/app/api/auth.py, python-backend/app/main.py</files>
  <action>
    1. Create python-backend/app/api/auth.py:
       - Create router = APIRouter()
       - POST /verify — accepts Firebase ID token in Authorization header, returns current user info. Uses Depends(get_current_user). Response: {"success": true, "user": {uid, email, display_name, role}}
       - GET /me — returns current authenticated user info. Uses Depends(get_current_user). Same response as /verify.
       - POST /set-role — CEO-only endpoint to set a user's role. Uses Depends(require_ceo). Accepts {uid: str, role: str} body. Calls firebase_admin.auth.set_custom_user_claims(uid, {"role": role}). Returns {"success": true, "message": "Role updated"}.
